api

uri를 접근하면 리턴하는 파일이 json방식으로 전달한다.

uri가 곧 api라고 할 수 있다.

uri안에 들어 있는 파라메터를 이용하여 프로그램을 완성해서

ResponseBody가 객체를 json형태로 바꿔준다 

리턴으로 던져줄 수 있다.

던지는 것이 rest-api일 수도 있고, ajax 던질 수 있다.
api로 던질 때는 보안을 고려해야한다.


파일을 직접 주지 않는다 json의 형태로 준다.

Uri로 협력업체가 들어와서 필요한 정보를 받아가서 협업한다.

프로그램의 직접적인 내용을 볼 수는 없다.

json을 파싱해서 본인들의 프로그램을 돌린다.


뷰를 부르는 것이 아니라 json을 던지면 api라고 한다.
